build step. This doc is the orientation for a fresh debugging session.

## Architecture

Single-page, no framework, no router. `STATE.view` + `render()` switch between views:
`home`, `workout`, `history`, `guide`, `settings`. `nav(view)` sets it and re-renders.

### STATE (in `app.js`)
```
maxes              { snatch, cj, bs, fs, pp, bench }  — 1RM training maxes
program            { blockId, weekInBlock }            — blockId 0=Testing,1,2,3,4=Deload; week is 0-indexed
cutting            bool   — deficit mode (reduces volume)
noSport            bool   — false = assume ~2 sport sessions (default); true = reinstate plyos + Wed intervals
log                { 'YYYY-MM-DD': { dayKey, title, setsLogged, sessionMin } }
hypertrophyWeights { exId: { weight, sets[], prevSets[], repTop, setCount, progressNext } }
restTimer / sessionTimer / intervalTimer / activeWorkout / wakeLock
```
- **Persistence:** `save()`/`load()` → localStorage key `oly_state`. Persists maxes, program, cutting, noSport, log, hypertrophyWeights. **`activeWorkout` is NOT persisted** — reloading mid-workout loses the in-progress session.

### Program data model (`program.js`)
- `EX` — exercise library keyed by id: `{ name, type, notes, cues[], baseLift }`. `type` ∈ oly|strength|hypertrophy|technical|cardio|core|jump|warmup|max_effort|mobility.
- `PROGRAM.blocks` — array; each block has `getDay(dayKey, weekInBlock)`.
  - Blocks 1 & 2 generated by `makeDays(B, weekIdx)` from progression tables `B1`/`B2`.
  - Block 0 (Testing) and Block 3 (Peaking) have their own day objects.
- A **day**: `{ title, totalMin, totalMinNoSport?, sections[], isRest?, isTesting?, note?, lifts? }`.
- A **section**: `{ title, color, note?, exercises[] }`. color ∈ gold|blue|green|red.
- An **exercise slot**: `{ id, sets|null, reps|repRange, pct, baseLift, rest(sec), isDailyMax?, isMaxEffort?, buildup?, buildupNote?, optional?, optNote?, interval?, note? }`.
- **Build-up ladders:** `rampOlyDaily(targetPct, reps)`, `rampOlySaturday(...)`, and fixed `RAMP_SAT_BS/PP/BENCH`. Each step `{ pct, reps, rest, top? }`. Rendered as tappable rows (tap starts that step's rest).
- `PROGRAM.calcWeight(maxes, baseLift, pct)` → rounds to nearest 2.5 lb.
- `PROGRAM.getWorkout(blockId, week, dayKey, cutting)` = `getDayWorkout` + `applyCutting`.
- `PROGRAM.applyCutting(day)` → deep-clones, drops hypertrophy to 2 sets (face_pull exempt), oly/strength/technical −1 set.

### Key app.js functions
- `dayFor(dayKey)` — resolves the current block/week day, cutting applied. Use this everywhere a day is needed.
- `computeSchedule(day)` — assigns `_startSec` to each exercise; anchors total to `dayEstMin(day)`; **skips `optional` items unless `STATE.noSport`**. Returns `{ totalSec, list }`, stored on `activeWorkout`.
- `exerciseRawSec(ex, def)` — per-exercise time estimate: build-ups sum their ladder; cardio/mobility/intervals use real duration; else `sets × (exec + rest)` with exec 20–40s by type.
- `dayEstMin(day)` — expected minutes. Uses `totalMinNoSport` when `noSport`; ×0.85 when `cutting`.
- **Timers:**
  - Rest timer → `#timer-overlay`, `startRestTimer(sec)`.
  - Session timer → counts **down** from `activeWorkout.totalSec`; overtime shows `+MM:SS` in red. `updateSessionTimerDisplay()` ticks 1/s and calls `highlightCurrentExercise(elapsed)` (adds `.ex-now` to the due card). Per-exercise "Start @" labels are **countdown values** = `totalSec − _startSec`.
  - Interval timer → `#interval-overlay`, full phase engine (`startIntervalTimer`, `advanceIntervalPhase`, `catchUpIntervalTimer` for backgrounding). Used by Wednesday VO₂max.
- **Double progression:** `recordHypertrophySet()` accumulates reps during a session; `finalizeHypertrophyProgression()` (called in `endWorkout`) sets `progressNext` if all sets hit `repTop`, snapshots `prevSets`, clears live `sets`. `shouldProgress(exId)` reads `progressNext` → drives the "⬆ INCREASE WEIGHT" banner.
- **Settings toggles:** `setCutting(bool)`, `setNoSport(bool)`, `updateMax`, `updateBlock`, `updateWeek`.

## Debugging gotchas (these will bite you — read first)

1. **Preview HTTP cache is stubborn.** `location.reload(true)` does NOT reliably bust the
   preview browser's cache, so edits to `js/*.js` may not appear. Reliable fix: **change the
   port** in `C:\.claude\launch.json` (e.g. 3143 → 3145), `preview_stop` then `preview_start`
   — a new origin has no cache. Also unregister the SW + clear caches in-page:
   `navigator.serviceWorker.getRegistrations().then(r=>r.forEach(x=>x.unregister()))` and
   `caches.keys().then(k=>k.forEach(x=>caches.delete(x)))`.
2. **`preview_screenshot` times out while any timer is running** (the 1/s or 200ms
   `setInterval` keeps the page from going idle). Freeze first: `clearInterval(STATE.sessionTimer.interval)`
   (and `intervalTimer.interval`) before screenshotting. **Verify via `preview_eval` DOM
   inspection instead — it's reliable when screenshots aren't.**
3. **`activeWorkout` isn't persisted** — a mid-session reload drops it. Re-start the workout.
4. **Log is keyed by calendar date** (`today()`), so doing two different days in one calendar
   day overwrites the first.
5. **Time estimates are heuristic** for set execution (20–40s/set) but use exact rest periods
   and are anchored to each day's documented `totalMin`. Daily-max/max-effort days are the
   least accurate (real attempt count is autoregulated). See the in-app Guide → "is this legit"
   reasoning, or just know: straight-set days are tight, max days are ballpark.

## Feature map (where things live)

- Week 0 testing flow → `TESTING_WEEK` in `program.js`; `renderTestingDay()` in `app.js`.
- Cutting transform → `applyCutting` (program.js); toggle `setCutting`.
- No-sport toggle → `optional` flags on box/broad jumps + Wed intervals; gated by `STATE.noSport` in `computeSchedule` + `renderExerciseCard` (`inactive` var). `totalMinNoSport` per day.
- Build-up ladders → generators in `program.js`; rendered in `renderExerciseCard` (`.buildup`).
- Countdown timer + start times → `updateSessionTimerDisplay`, `computeSchedule`, `.ex-start` / `.ex-now`.
- Guide tab → `renderGuide()` (full program reference: rules, blocks, cutting, cardio, mobility, nutrition, lifestyle/autoregulation, coaching).
- Home day list (tap any day, shows focus + est time) → `renderHome()`.
